# Inline-цитаты в стиле Perplexity

**Дата:** 2026-05-22
**Компонент:** `client/src/utils/citations.ts`, рендер markdown в `MessageList` (`@gravity-ui/aikit`)

## Проблема

Сейчас inline-цитаты в ответе ассистента отображаются как markdown-ссылки `[1]`, `[2]`, `[3]` — синие, подчёркнутые, ведут на якорь `#src-<messageId>-<n>` внутри блока источников. Не нравится:

- Цвет ссылок выбивается из текста (выглядит как обычные внешние линки)
- Формат `[N]` обезличенный — пользователь не понимает, на какой источник ссылается фрагмент, не дочитав до блока источников

## Ожидаемое поведение (как в Perplexity)

После цитируемого фрагмента — небольшая **«пилюля» с доменом источника** (например `perplexity`, `wikipedia`, `practicum.yandex`). Стиль:

- Прямоугольный chip с фоном `var(--g-color-base-misc-light)` / monospace или близкий шрифт
- Текст — `caption-1`, secondary
- Кликабельный, ведёт на тот же якорь / открывает источник в новой вкладке
- При наличии нескольких источников подряд — `perplexity +1`, `perplexity +2` (как у Perplexity на скриншоте)

## План

- [x] В `citations.ts` заменить `injectCitationLinks` (выбран **вариант B** — markdown-ссылка с особым текстом + CSS-селектор по `href^="#src-"`)
- [x] Группировка цитат — все цитаты одного параграфа сливаются в **одну пилюлю в конце параграфа** (`домен +K`), как в Perplexity
- [x] ~~Hover-стейт пилюли~~ — по решению пользователя hover-смена цвета убрана
- [x] Тёмная тема — фон `--g-color-base-misc-medium` (alpha 0.5) даёт хороший контраст в обеих темах
- [x] Связь «пилюля → источник» через `window.open(url, '_blank')` (клик открывает URL источника в новой вкладке); якорь `#src-<messageId>-<n>` остаётся как маркер для CSS и парсинга href

## Открытые вопросы

- Что показывает пилюля если у источника нет читаемого хоста? (например, fallback на `s.url` — обрезать?)
- Как разрешать `aikit` рендерить кастомный markdown-узел? Возможно, придётся пройти по дереву AST вручную или использовать собственный рендерер.
- Поведение на streaming: пилюли подсвечиваются по мере прихода `[N]` в тексте — нужно убедиться, что подстановка работает с `shouldParseIncompleteMarkdown`.

## Acceptance Criteria

- [x] Вместо `[1]`, `[2]` в тексте — пилюли с доменом источника, без подчёркивания, не «синие как ссылка»
- [x] Клик по пилюле — открывает URL источника в новой вкладке (`target="_blank"`, через `window.open`)
- [x] Несколько подряд идущих цитат сгруппированы в одну пилюлю с `+N` (теперь — все цитаты параграфа сливаются в одну пилюлю в конце параграфа)
- [x] Не сломан существующий парсинг markdown в ответе
