# Результат: реальные вход/исходящие токены (usage)

Реализовано полностью, на компонентах AIKit 2.x. Эвристика `length/3` осталась только как прогноз контекста (`ContextIndicator`), фактический расход теперь берётся из `usage` LLM.

## Что сделано

**Бэкенд**
- `streamChat` запрашивает `stream_options: { include_usage: true }`.
- В цикле стрима копится `usage` суммарно по всем tool-раундам; сохраняется в `messages.prompt_tokens` / `completion_tokens`; уходит в SSE-событие `done` (`usage`).
- Миграция: две новые колонки (`CREATE TABLE` + `ALTER` для существующих БД).

**Типы** — `Message` и событие `done` в `shared/types.ts` расширены полями токенов.

**Фронтенд**
- `usage` прокинут через `onDone` → стор → `message.metadata`.
- Под каждым ответом — `ContextItem` `↑prompt ↓completion` через `MessageList.assistantExtraInfo`.
- В шапке композера — итог по чату `Σ ↑.. ↓..` (`ContextItem`) рядом с `ContextIndicator`.

## Проверка

- `tsc --noEmit` (client + server) — чисто.
- Браузерный smoke (Playwright, чистый dev-сервер):
  - живой стрим: под ответом `↑518 ↓38`, в композере `Σ ↑518 ↓38`;
  - загрузка существующего чата (История → клик): токены приходят из БД, отображаются;
  - 0 ошибок консоли.
- БД/API проверены напрямую: `prompt_tokens`/`completion_tokens` сохраняются и отдаются GET-эндпоинтом.

## Замечание (вне рамок задачи)

Обнаружен **pre-existing баг**: hard-reload по прямому URL `/chat/:id` рендерит пустой список сообщений (гонка в `ChatPage`/`chatStore` — `setActive` сбрасывает `messages: []`, двойной `fetchChats` под React StrictMode её провоцирует). Нормальный путь (навигация через Историю) работает. Файлы этой фичей не затрагивались — стоит завести отдельную задачу.
