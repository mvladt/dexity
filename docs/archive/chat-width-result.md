# Result: Изъяны ширины компонента чата

**Дата:** 2026-05-12
**Связан с:** `docs/chat-width-task.md`

## Что изменено

**`client/src/styles.css`** — единственный изменённый файл:

1. `--chat-max-width: 860px` добавлена как CSS-переменная на `.chat-layout`.
2. `.chat-content` — убраны `max-width: 860px` и `margin-inline: auto`. Контейнер теперь full-width внутри `.chat-main`.
3. `.chat-main-header`, `.chat-messages`, `.chat-input` — `padding: <block> <inline>` заменено на `padding-block` + `padding-inline: max(16px, calc((100% - var(--chat-max-width)) / 2 + 16px))`.

Формула `max(16px, calc((100% - 860px) / 2 + 16px))` вычисляет боковые отступы так, что содержимое центрируется в 860 px, а при ширине меньше 860 px автоматически деградирует до 16 px.

## Подход

Центрирование через `padding-inline` на full-width контейнерах — вместо `max-width` + `margin: auto` на вложенном блоке. Это стандартный паттерн ChatGPT/Claude/Perplexity: прокрутка, хедер и инпут-бар занимают всю ширину области, а контент центрируется через отступы внутри них. JSX-компоненты не тронуты.
