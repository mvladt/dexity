## BUG-001 — [CRITICAL] Нет поля ввода на главном экране `/chat`

**Сценарий:** Пользователь открывает приложение → попадает на `/chat`.

**Ожидаемое поведение:** Есть возможность ввести произвольное сообщение.

**Фактическое поведение:** Отображаются приветственный заголовок "Чем могу помочь?" и кнопки быстрых действий, но поля ввода нет вообще. Пользователь не может начать новый диалог со своим вопросом.

**Workaround:** Нажать "+ Новый" → перейти на `/chat/:id` где поле ввода есть.

**Скриншот:** `screenshots/01-initial-load.png`

## BUG-002 — [MEDIUM] Горизонтальный скролл в сайдбаре

**Сценарий:** Любая страница с сайдбаром.

**Фактическое поведение:** У сайдбара появляется горизонтальная полоса прокрутки. Скролл лишний — сайдбар не должен скролиться горизонтально, его содержимое должно умещаться по ширине.

**Вероятная причина:** Какой-то дочерний элемент (возможно `.gravity-ui` компонент) выходит за ширину сайдбара.

**Скриншот:** `screenshots/03-existing-chat.png`, `screenshots/06-new-chat.png`

## BUG-003 — [LOW] Placeholder поля ввода на английском

**Сценарий:** Открыть любой чат `/chat/:id`.

**Фактическое поведение:** В поле ввода отображается placeholder **"Plan, code, build and test anything"** (английский). Всё приложение на русском, это выбивается.

**Ожидаемое поведение:** Placeholder на русском, например "Напишите сообщение…".

**Скриншот:** `screenshots/03-existing-chat.png`

## BUG-004 — [INFO] 20 console warnings при загрузке страницы

**Источник:** `@gravity-ui/aikit`

**Суть:** Библиотека пытается обратиться к Node.js-модулям (`process`, `path`, `fs`, `url`, `source-map-js`) в контексте браузера. Vite их экстернализирует и выдаёт warnings.

**Влияние:** Функциональность не нарушается, но консоль засорена. Предупреждения — на стороне библиотеки, не приложения.

```
Module "process" has been externalized for browser compatibility...
Module "path" has been externalized for browser compatibility...
Module "fs" has been externalized for browser compatibility...
...
```
