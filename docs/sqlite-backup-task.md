# Бэкап SQLite на проде — Task

## Контекст

В `docs/archive/vps-deploy-cicd-plan.md` (риск 5) было зафиксировано решение вынести БД за пределы
релизов (`/srv/dexity/data/db.sqlite3`) именно ради бэкапов — но сам cron-джоб так и не завёлся.
Прод (`dexity.mvladt.ru`) сейчас работает без резервного копирования БД.

## Что сделать

- [ ] Скрипт бэкапа на spb: `sqlite3 /srv/dexity/data/db.sqlite3 ".backup '/srv/dexity/backups/db-$(date +%F).sqlite3'"`
      (`.backup` — безопасно при активном WAL, не требует останавливать `dexity-server`)
- [ ] Ротация — хранить последние N бэкапов (7? 30?), старые удалять
- [ ] cron на spb (например, ежедневно ночью)
- [ ] Проверить: бэкап реально восстанавливается (`sqlite3 backup.sqlite3 ".tables"` или пробный запуск сервера на копии)
- [ ] Решить, где хранить бэкапы: только локально на spb, или дополнительно скачивать за пределы
      сервера (на случай потери самого VPS)
